はじめかた

Conductor キーボードにマクロを追加し、カスタムファームウェアとしてビルド・書き込みするための準備手順です。所要時間は約 5 分です。

1

GitHub アカウントを作る

すでにアカウントをお持ちの方はスキップしてください。
github.com/signup から無料で作成できます。

2

公式リポジトリをフォークする

まず 公式リポジトリのブランチ一覧 を開き、使いたいバージョンのブランチ名を確認してください（例: release/0.6.3）。

次に リポジトリのトップページ に戻り、右上の Fork ボタンを押してください。

画面右上にある Fork ボタン → Create fork をクリック

これであなたのアカウントにリポジトリのコピーが作られます。マクロの変更はこのコピーに保存されるので、公式リポジトリには影響しません。

3

自動ビルドを有効にする

フォークしたリポジトリを開き、上部のタブから Actions をクリックしてください。

「I understand my workflows, go ahead and enable them」
と表示されたら、緑色のボタンを押して有効化

これでマクロを変更するたびに、ファームウェアが自動でビルドされるようになります。

この手順は初回のみです。一度有効化すれば、次回以降は不要です。
4

Macro Editor にログイン

Conductor Macro Editor に戻り、GitHub でログイン ボタンを押してください。

認証画面が開くので、表示されたコードを入力して許可すると自動的にログインされます。

5

マクロを作成してデプロイ

ログイン後、リポジトリとブランチを選んで Fetch を押すと keymap が読み込まれます。

左パネルの + New でマクロを作成し、右のキーボード表示でキーに割り当てたら、下の Commit & Push を押すだけ。自動ビルドが始まります。

ビルド完了後、フォーク先リポジトリの Actions タブから firmware をダウンロードし、キーボードに書き込んでください。

Conductor Macro Editor を開く

準備ができたら、さっそく始めましょう
